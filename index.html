1. Wyjaśnienie nagłówka Window

Każdy plik XAML zaczyna się od definicji kontenera. Oto co oznaczają poszczególne atrybuty:

  • x:Class="wpf_nauka.MainWindow" – Łączy plik XAML z plikiem C# (MainWindow.xaml.cs). Mówi kompilatorowi: "Logika tego okna jest w klasie MainWindow w przestrzeni wpf_nauka".
  • xmlns="..." – (XML Namespace). Standardowy adres, który mówi, że używamy kontrolek WPF (Button, Grid itp.).
  • xmlns:x="..." – Reguły XAML (np. nadawanie nazw kontrolkom przez x:Name).
  • xmlns:d i xmlns:mc – Ustawienia tylko dla projektanta (Design-time). Pozwalają np. widzieć dane w podglądzie Visual Studio, które nie pojawią się w gotowej aplikacji.
  • xmlns:local="..." – Pozwala używać Twoich własnych klas C# wewnątrz XAML.
  • mc:Ignorable="d" – Mówi: "Zignoruj wszystko co zaczyna się od 'd:' podczas uruchamiania programu (używaj tylko w projektancie)".

2. Przydatne ustawienia Window (Snippets)

Wklej te atrybuty do tagu <Window ... >, aby szybko zmienić zachowanie aplikacji: 

Title="Moja Aplikacja"       Height="450" Width="800"      WindowStartupLocation="CenterScreen" ResizeMode="NoResize"         Topmost="True"                Background="#F5F5F5"          Icon="icon.ico"

3. Kluczowe zdarzenia (Events) Okna

Zdarzenia, które warto znać przy inicjalizacji danych:

<Window ...
    Loaded="Window_Loaded"      Closing="Window_Closing"    KeyDown="Window_KeyDown">   </Window>

2.1. Głębokie nurkowanie w Atrybuty Okna

Poniżej znajdziesz detale techniczne, które tłumaczą, dlaczego Twój projekt w ogóle widzi kontrolki WPF.

1. xmlns (Default Namespace): xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
To jest "mapa drogowa" dla standardowych kontrolek. Bez tego tag <Button> byłby zwykłym tekstem, a nie przyciskiem. Mapuje on tagi XAML na klasy w bibliotekach .NET (System.Windows.Controls).
2. xmlns:x (XAML Namespace): xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
Daje dostęp do funkcji samego języka XAML.
  • x:Name – nadaje nazwę zmiennej dostępną w C#.
  • x:Key – unikalny identyfikator w zasobach (Resources).
  • x:Static – pozwala odwołać się do stałych w C#.
3. Design-Time Attributes: xmlns:d, xmlns:mc oraz mc:Ignorable="d"
Te linie to "układ" z edytorem wizualnym Visual Studio.
  • Pozwalają używać atrybutów typu d:DataContext lub d:DesignHeight.
  • Zaleta: Możesz ustawić, żeby w edytorze okno miało 800px wysokości (by wygodnie projektować), ale po uruchomieniu programu miało 300px.
  • mc:Ignorable="d" mówi programowi: "Podczas kompilacji wyrzuć wszystko, co zaczyna się na literę d".
4. xmlns:local: xmlns:local="clr-namespace:wpf_nauka"
To jest Twój "most" do własnego kodu.
  • clr-namespace oznacza "Common Language Runtime namespace".
  • Pozwala Ci używać Twoich własnych klas w XAML, np. <local:MojViewModel />.
  • Jeśli stworzysz folder Converters, musisz dopisać: xmlns:conv="clr-namespace:wpf_nauka.Converters".

3. Grid - Król Layoutów

Grid pozwala dzielić okno na wiersze (Rows) i kolumny (Columns). Elementy umieszczasz w nich za pomocą indeksów (liczonych od 0).

Kluczowe jednostki miary:
  • 100 (Piksele) – stały rozmiar, nie zmienia się przy skalowaniu okna.
  • Auto – rozmiar dopasowuje się do zawartości (np. szerokość przycisku zależy od długości tekstu).
  • * (Star / Gwiazdka) – zajmuje całą dostępną przestrzeń. Jeśli masz dwa wiersze *, każdy zajmie 50%. Jeśli jeden ma 2* a drugi *, pierwszy zajmie 2/3 miejsca.

3.1. Szablon struktury Grida

<Grid>
    <!-- Definicja kolumn -->
    <Grid.ColumnDefinitions>
        <ColumnDefinition Width="200"/> <!-- Stała szerokość (np. menu boczne) -->
        <ColumnDefinition Width="*"/>   <!-- Reszta miejsca -->
    </Grid.ColumnDefinitions>

    <!-- Definicja wierszy -->
    <Grid.RowDefinitions>
        <RowDefinition Height="Auto"/> <!-- Wysokość zależna od tekstu -->
        <RowDefinition Height="*"/>    <!-- Reszta miejsca -->
    </Grid.RowDefinitions>

    <!-- Rozmieszczenie kontrolek -->
    <TextBlock Grid.Row="0" Grid.Column="0" Text="Logo" />
    <Button Grid.Row="0" Grid.Column="1" Content="Górny przycisk" />
    
    <!-- Element rozciągnięty na dwie kolumny (ColumnSpan) -->
    <Rectangle Grid.Row="1" Grid.Column="0" Grid.ColumnSpan="2" Fill="LightBlue" />
</Grid>
Trick na laboratoria: Jeśli zapomnisz dopisać Grid.Row lub Grid.Column, WPF domyślnie wstawi kontrolkę do komórki (0,0). Jeśli wstawisz kilka kontrolek do tej samej komórki, będą one leżeć jedna na drugiej (ostatnia na wierzchu).

3.2. GridSplitter - Zmiana rozmiaru myszką

Chcesz, żeby użytkownik mógł sam przesuwać granicę między kolumnami? Użyj GridSplittera. 

<!-- Wstaw go jako osobną kolumnę o małej szerokości lub na krawędzi -->
<GridSplitter Grid.Column="0" Width="5" HorizontalAlignment="Right" VerticalAlignment="Stretch" Background="Gray"/>

4.1. Klasyczne Menu Górne (Menu Bar)

Standardowe menu z rozwijanymi opcjami (Plik, Edycja, Pomoc). Używa się go zazwyczaj z DockPanel. 

<DockPanel>
    <Menu DockPanel.Dock="Top">
        <MenuItem Header="_Plik"> <!-- Podkreślnik przed literą to skrót Alt+P -->
            <MenuItem Header="_Nowy" Command="{Binding NewCommand}" />
            <MenuItem Header="_Otwórz" Click="Open_Click" />
            <Separator />
            <MenuItem Header="Wyjście" InputGestureText="Ctrl+E" Click="Exit_Click" />
        </MenuItem>
        <MenuItem Header="Opcje">
            <MenuItem Header="Pokaż pasek statusu" IsCheckable="True" IsChecked="True" />
        </MenuItem>
    </Menu>
    
    <!-- Reszta treści pod menu -->
    <Grid> ... </Grid>
</DockPanel>

4.2. Nowoczesne Menu Boczne (Side Menu)

Wykorzystuje Grid z kolumną o stałej szerokości i przyciski stylizowane na menu. 

<Grid>
    <Grid.ColumnDefinitions>
        <ColumnDefinition Width="200"/> <!-- Kolumna menu -->
        <ColumnDefinition Width="*"/>   <!-- Główna treść -->
    </Grid.ColumnDefinitions>

    <!-- Panel menu bocznego -->
    <StackPanel Grid.Column="0" Background="#2c3e50">
        <TextBlock Text="MOJA APKA" Foreground="White" Margin="10" FontSize="18" HorizontalAlignment="Center"/>
        
        <Button Content="Dashboard" Style="{StaticResource MenuButtonStyle}" />
        <Button Content="Użytkownicy" Style="{StaticResource MenuButtonStyle}" />
        <Button Content="Ustawienia" Style="{StaticResource MenuButtonStyle}" />
    </StackPanel>

    <!-- Obszar na treść (np. Frame do nawigacji między stronami) -->
    <Frame Grid.Column="1" Name="MainFrame" NavigationUIVisibility="Hidden" />
</Grid>

4.3. Menu Kontekstowe (Prawy Przycisk Myszy)

Możesz przypisać menu do niemal każdej kontrolki (np. Buttona lub Listy). 

<Button Content="Kliknij prawym">
    <Button.ContextMenu>
        <ContextMenu>
            <MenuItem Header="Kopiuj" Click="Copy_Click" />
            <MenuItem Header="Usuń" Click="Delete_Click" Foreground="Red" />
        </ContextMenu>
    </Button.ContextMenu>
</Button>

4.4. Jak ożywić menu? (Obsługa zdarzeń)

Istnieją dwa sposoby na sprawienie, by kliknięcie w menu coś robiło:

Metoda 1: Event Handler (Szybka)

Najprostsza na labach. W XAML dodajesz Click, a w C# piszesz kod. 

<!-- MainWindow.xaml -->
<MenuItem Header="Zamknij" Click="Exit_Click" />
// MainWindow.xaml.cs
private void Exit_Click(object sender, RoutedEventArgs e)
{
    Application.Current.Shutdown(); // Zamyka całą aplikację
}

Metoda 2: Command (MVVM)

Zalecana w większych projektach. Oddziela wygląd od logiki.

<!-- MainWindow.xaml -->
<MenuItem Header="Otwórz" Command="{Binding OpenFileCommand}" />
Pamiętaj: Aby Command działał, Twój DataContext musi być ustawiony na klasę ViewModel, która zawiera definicję OpenFileCommand.

4.5. Nawigacja - przełączanie widoków

Jeśli menu ma zmieniać treść na środku ekranu, użyj kontrolki Frame. 

<!-- 1. W MainWindow.xaml umieść Frame w odpowiednim miejscu Grida -->
<Frame x:Name="MainContentFrame" NavigationUIVisibility="Hidden" />

<!-- 2. W menu dodaj zdarzenie Click -->
<MenuItem Header="Ustawienia" Click="GoToSettings_Click" />
// 3. W MainWindow.xaml.cs (zakładając, że masz plik SettingsPage.xaml)
private void GoToSettings_Click(object sender, RoutedEventArgs e)
{
    MainContentFrame.Navigate(new Uri("SettingsPage.xaml", UriKind.Relative));
}

5. Data Binding - Synchronizacja Danych

Binding łączy właściwość kontrolki (np. Text) ze źródłem danych (np. zmienną w C#).

Złota zasada: Aby Binding działał, musisz ustawić DataContext. Zazwyczaj w konstruktorze okna (MainWindow.xaml.cs) piszesz: this.DataContext = this; (jeśli dane są w tym samym pliku) lub this.DataContext = new MyViewModel();.

5.1. Tryby bindowania (Mode)

  • OneWay – Dane idą tylko ze źródła do UI (np. wyświetlanie statusu).
  • TwoWay – Zmiana w UI zmienia dane w C# i odwrotnie (np. pola tekstowe).
  • OneTime – Dane ładują się tylko raz przy starcie (oszczędność wydajności). 
<!-- Przykład TwoWay z natychmiastową aktualizacją -->
<TextBox Text="{Binding NazwaUzytkownika, Mode=TwoWay, UpdateSourceTrigger=PropertyChanged}" />

5.2. Zaawansowane: StringFormat i FallbackValue

Możesz formatować tekst bezpośrednio w XAML bez pisania kodu C#.

<!-- Wyświetlanie ceny z walutą i wartością domyślną gdy danych brak -->
<TextBlock Text="{Binding Cena, StringFormat='Cena: {0:C}', FallbackValue='Brak ceny'}" />

<!-- Formatowanie daty -->
<TextBlock Text="{Binding DataUtworzenia, StringFormat='{}Data: {0:dd.MM.yyyy}'}" />

5.3. IValueConverter - Transformacja danych

Używasz go, gdy dane w C# są innego typu niż wymaga kontrolka (np. bool w C#, a chcesz zmienić Kolor tła). 

// 1. Klasa konwertera w C#
public class BoolToColorConverter : IValueConverter
{
    public object Convert(object value, Type targetType, object parameter, CultureInfo culture)
    {
        return (bool)value ? Brushes.Green : Brushes.Red;
    }
    public object ConvertBack(object value, Type targetType, object parameter, CultureInfo culture) => throw new NotImplementedException();
}
<!-- 2. Rejestracja w XAML (Window.Resources) -->
<Window.Resources>
    <local:BoolToColorConverter x:Key="MyColorConv" />
</Window.Resources>

<!-- 3. Użycie -->
<Ellipse Fill="{Binding CzyAktywny, Converter={StaticResource MyColorConv}}" Width="50" Height="50" />

5.4. MultiBinding (Zadanie na 5.0)

Gdy jeden tekst zależy od kilku zmiennych (np. Imię + Nazwisko).

<TextBlock>
    <TextBlock.Text>
        <MultiBinding StringFormat="{}{0} {1}">
            <Binding Path="Imie" />
            <Binding Path="Nazwisko" />
        </MultiBinding>
    </TextBlock.Text>
</TextBlock>

5.5. Proste Bindowania (Interakcja UI-UI)

Te techniki działają całkowicie w XAML. Nie wymagają ViewModelu ani INotifyPropertyChanged.

ElementName: Pozwala na pobranie wartości z innej kontrolki za pomocą jej nazwy (x:Name).

A. Bindowanie Suwaka do Tekstu

Idealne, gdy chcesz pokazać wartość liczbową (np. głośność, wiek).

<Slider x:Name="MySlider" Minimum="0" Maximum="100" />
<TextBlock Text="{Binding Value, ElementName=MySlider, StringFormat='Wartość: {0:F0}'}" />

B. Włączanie/Wyłączanie przycisku przez CheckBox

Przycisk będzie aktywny tylko wtedy, gdy użytkownik zaznaczy "Akceptuję regulamin". 

<CheckBox x:Name="AcceptTerms" Content="Akceptuję warunki" />
<Button Content="Zarejestruj" IsEnabled="{Binding IsChecked, ElementName=AcceptTerms}" />

C. Synchronizacja pól tekstowych

To co wpiszesz w jednym TextBoxie, natychmiast pojawia się w innym elemencie. 

<TextBox x:Name="InputSource" />
<TextBlock Text="{Binding Text, ElementName=InputSource}" Foreground="Blue" />

D. Bindowanie Koloru (Wybór z Listy)

Pozwala użytkownikowi wybrać kolor tła z listy (ComboBox).

<ComboBox x:Name="ColorPicker" SelectedIndex="0">
    <ComboBoxItem Content="Red"/>
    <ComboBoxItem Content="Green"/>
    <ComboBoxItem Content="Blue"/>
</ComboBox>

<Rectangle Width="100" Height="20" 
           Fill="{Binding SelectedItem.Content, ElementName=ColorPicker}" />

E. Ukrywanie elementów (Visibility)

Pokazuje kontrolkę tylko wtedy, gdy w innym polu jest tekst (używamy wbudowanego konwertera). 

<Window.Resources>
    <BooleanToVisibilityConverter x:Key="BoolToVis" />
</Window.Resources>

<ToggleButton x:Name="ShowExtra" Content="Pokaż więcej" />
<StackPanel Visibility="{Binding IsChecked, ElementName=ShowExtra, Converter={StaticResource BoolToVis}}">
    <TextBlock Text="To jest dodatkowa treść!" />
</StackPanel>

5.6. Bindowanie do C# (Model-View-ViewModel)

Aby UI reagowało na zmiany w kodzie C#, klasa musi informować system: "Hej, zmieniłem wartość, odśwież się!".

A. Standardowy "BaseViewModel" (Kopiuj-Wklej)

Stwórz klasę ViewModelBase.cs i używaj jej jako bazy dla swoich danych. 

using System.ComponentModel;
using System.Runtime.CompilerServices;

public class ViewModelBase : INotifyPropertyChanged
{
    public event PropertyChangedEventHandler PropertyChanged;

    protected void OnPropertyChanged([CallerMemberName] string propertyName = null)
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
    }
}

B. Tworzenie właściwości z powiadomieniem

Zamiast zwykłego public string Name {get; set;}, musisz użyć pełnej formy: 

public class MainViewModel : ViewModelBase
{
    private string _userName = "Jan";
    
    public string UserName
    {
        get => _userName;
        set 
        { 
            _userName = value; 
            OnPropertyChanged(); // To jest klucz do odświeżania UI!
        }
    }
}

C. Podpięcie pod UI (MainWindow.xaml.cs)

Musisz powiedzieć oknu, skąd ma brać dane. Robi się to w konstruktorze. 

public partial class MainWindow : Window
{
    public MainWindow()
    {
        InitializeComponent();
        // Ustawiamy źródło danych dla całego okna
        this.DataContext = new MainViewModel();
    }
}

D. Użycie w XAML

<!-- Teraz "UserName" odnosi się bezpośrednio do właściwości w klasie MainViewModel -->
<TextBox Text="{Binding UserName, UpdateSourceTrigger=PropertyChanged}" />
<TextBlock Text="{Binding UserName, StringFormat='Witaj, {0}!'}" />

5.7. Bindowanie do Kolekcji (Listy)

Użyj ObservableCollection, aby lista w UI sama się aktualizowała po dodaniu/usunięciu elementu w C#. 

// W ViewModelu:
public ObservableCollection<string> Tasks { get; set; } = new ObservableCollection<string>();

// Metoda dodająca:
Tasks.Add("Nowe zadanie do zrobienia");
<!-- W XAML -->
<ListBox ItemsSource="{Binding Tasks}" />

<!-- Lub bardziej zaawansowany ComboBox -->
<ComboBox ItemsSource="{Binding Tasks}" SelectedIndex="0" />

6. Window.Resources - Magazyn Projektów

Sekcja <Window.Resources> służy do definiowania obiektów, które chcesz wykorzystać w wielu miejscach okna.

StaticResource vs DynamicResource:
  • StaticResource – Wartość jest pobierana raz, przy starcie okna (najszybsza, używaj jej do 90% przypadków).
  • DynamicResource – Wartość może się zmienić w trakcie działania programu (używane np. do zmiany motywu z jasnego na ciemny "w locie").

6.1. Definiowanie Kolorów i Pędzli

Zamiast wpisywać kolor ręcznie w każdym przycisku, zdefiniuj go raz. 

<Window.Resources>
    <SolidColorBrush x:Key="PrimaryColor" Color="#0078d4" />
    <SolidColorBrush x:Key="AccentColor" Color="#ffb900" />
</Window.Resources>

<!-- Użycie -->
<Button Background="{StaticResource PrimaryColor}" Content="OK" />
<Button Background="{StaticResource AccentColor}" Content="Anuluj" />

6.2. Style (CSS dla WPF)

Styl pozwala nałożyć zestaw cech na wiele kontrolek naraz. Jeśli nie nadasz x:Key, styl zostanie nałożony na wszystkie kontrolki danego typu w oknie. 

<Window.Resources>
    <!-- Styl dla wszystkich przycisków w oknie -->
    <Style TargetType="Button">
        <Setter Property="Padding" Value="10,5" />
        <Setter Property="FontSize" Value="14" />
        <Setter Property="FontWeight" Value="Bold" />
    </Style>
</Window.Resources>

6.3. Rejestracja Konwerterów i ViewModeli

To tutaj musisz "zameldować" swoje klasy C#, aby XAML mógł je widzieć. 

<Window.Resources>
    <!-- Rejestrujemy konwerter (wymaga xmlns:local na górze) -->
    <local:MyValueConverter x:Key="MyConverter" />
    
    <!-- Można tu też stworzyć instancję ViewModelu (Design-time data) -->
    <local:MainViewModel x:Key="MyVM" />
</Window.Resources>

7. Kompletny Moduł: Lista z Dodawaniem (Lab 1)

Przykład zawiera: ObservableCollection, DataTemplate w ListBoxie oraz otwieranie nowego okna.

7.1. Widok (MainWindow.xaml)

<!-- Podstawowa struktura okna -->
<Window x:Class="Wpf_lab111.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        Title="MainWindow" Height="450" Width="800">
    <Grid>
        <DockPanel Margin="20">
            <!-- Panel górny do wpisywania danych -->
            <StackPanel DockPanel.Dock="Top" Orientation="Horizontal" Margin="0,0,0,20">
                <TextBox x:Name="txtInput" Width="200" Margin="0,0,10,0" VerticalContentAlignment="Center"/>
                <Button Content="Dodaj do listy" Click="AddBtn_Click" Padding="10,5"/>
                <Button Content="Otwórz nowe okno" Click="OpenNewWindow_Click" Margin="10,0"/>
            </StackPanel>

            <!-- Lista z własnym wyglądem każdego elementu -->
            <ListBox ItemsSource="{Binding Users}">
                <ListBox.ItemTemplate>
                    <DataTemplate>
                        <StackPanel Orientation="Horizontal" Margin="5">
                            <CheckBox IsChecked="{Binding IsSelected}" VerticalAlignment="Center" Margin="0,0,10,0"/>
                            <TextBox Text="{Binding Name}" FontSize="16" VerticalAlignment="Center"/>
                        </StackPanel>
                    </DataTemplate>
                </ListBox.ItemTemplate>
            </ListBox>
        </DockPanel>
    </Grid>
</Window>

7.2. Logika (MainWindow.xaml.cs)

namespace Wpf_lab111
{
    public partial class MainWindow : Window
    {
        // Klasa danych (Model)
        public class Userr
        {
            public string Name { get; set; }
            public bool IsSelected { get; set; } 

            public Userr(string name, bool isSelected = false)
            {
                Name = name;
                IsSelected = isSelected;
            }
        }

        // Kolekcja obserwowalna - UI samo wykryje dodanie/usuniecie elementu
        public ObservableCollection<Userr> Users { get; set; }

        public MainWindow()
        {
            Users = new ObservableCollection<Userr>();
            InitializeComponent();
            this.DataContext = this; // Ustawienie źródła bindowania na tę klasę
        }

        // Obsługa dodawania użytkownika
        private void AddBtn_Click(object sender, RoutedEventArgs e)
        {
            if (!string.IsNullOrWhiteSpace(txtInput.Text))
            {
                Users.Add(new Userr(txtInput.Text));
                txtInput.Clear();
            }
        }

        // Otwieranie nowego okna (ShowDialog blokuje okno główne)
        private void OpenNewWindow_Click(object sender, RoutedEventArgs e)
        {
            MainWindow win1 = new MainWindow();
            win1.ShowDialog(); 
        }
    }
}
Dlaczego to działa? (Analiza):
  • this.DataContext = this; – Dzięki temu {Binding Users} w XAML wie, że ma szukać właściwości Users w pliku .cs.
  • ObservableCollection – To specjalna lista. Gdy robisz Users.Add(), UI automatycznie dorysowuje nowy element w ListBoxie.
  • ItemTemplate – Pozwala "zaprojektować" jak ma wyglądać jeden wiersz na liście (u Ciebie to CheckBox + TextBox).
  • ShowDialog() – Otwiera nowe okno w trybie modalnym. Dopóki go nie zamkniesz, nie możesz kliknąć w okno "rodzica".
